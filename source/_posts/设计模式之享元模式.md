## Flyweight
属于 对象性能 模式

当系统充斥着大量的细粒度对象时 会带来 大量的开销 特别是内存

运用共享技术 有效地支持大量细粒度的对象

举个例子 在某个系统中 需要使用字体 我们为其实现一个字体对象 但是 字体对象很大 而且 字体 有可能相同 因此不需要创建多次相同的字体对象 而是将其存入一个对象池

<!-- more -->

```c++
class Font {
private:
    string key;
    /* ... */
public:
    Font(const string &key) { /* */ }
};
class FontFactory {
private:
    // 字体池 用来共享字体对象 没必要为相同的字体反复创建对象
    map<string, Font*> fontPool;
public:
    Font *GetFont(const string &key) {
        map<string, Font*>::iterator item = fontPool.find(key);
        // 找得到就返回 找不到就存入对象池
        if(item != footPool.end()) {
            return fontPool[key];
        }
        else {
            Font *font = new Font(key);
            fontPool[key] = font;
            return font;
        }
    }
};
```

缺陷 这种方式创建的对象是 只读的 不能改变其状态
