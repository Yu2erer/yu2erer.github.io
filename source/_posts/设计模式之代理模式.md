## Proxy
属于 接口隔离 模式

在面向对象系统中 有些对象由于某种原因(对象创建开销很大 或者 某些操作需要安全控制 或者 需要进程外的访问等) 直接访问会给使用者 或者系统结构带来很多麻烦

为其他对象提供一种代理以控制(隔离: 使用接口) 对这个对象的访问

举个超级简单的例子 有个客户端 要去做某个操作

<!-- more -->

```c++
class ISubject {
public:
    virtual void process();
};
class RealSubject : public ISubject {
public:
    virtual void process() { /* */ }
};
class ClientApp {
public:
    ISubject *subject;
    ClientApp() { subject = new RealSubject(); }
    
    void DoTask() {
        /* */
        subject->process();
    }
};
```

上面看起来已经完成了 但是那个对象 出于某种原因 不能被直接访问到 于是就要加 一个 proxy层 去间接访问 那个对象

```c++
class ISubject {
public:
    virtual void process();
};
// Proxy
class SubjectProxy : public ISubject {
public:
    virtual void process() {
        // 对RealSubject的一种间接访问
    }
};
class ClientApp {
public:
    ISubject *subject;
    ClientApp() { subject = new SubjectProxy(); }
    
    void DoTask() {
        /* */
        subject->process();
    }
};
```
