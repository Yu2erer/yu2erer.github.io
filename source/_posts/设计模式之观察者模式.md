## Observer
可分为 行为型模式 或者 组件协作模式

定义对象一对多(变化)的依赖关系 以便当一个对象的状态发生改变时 所有依赖于它的对象都得到通知并自动更新

假设现在有一个文件分割的工具 现在要加入一个新需求 在文件分割的时候 提供一个进度条显示进度

下面是代码例子

```c++
// 界面
class MainForm : public Form {
    TextBox* txtFilePath;
    TextBox* txtFileNumber;

    ProgressBar* progressBar;
public:
    void Button1_Click(){
        string filePath = txtFilePath->getText();
        int number = atoi(txtFileNumber->getText().c_str());
        FileSplitter splitter(filePath, number, progressBar);
        splitter.split();
    }
};
// 文件分割器
class FileSplitter {
    string m_filePath;
    int m_fileNumber;

    // 不能保证进度通知一定是通过进度条表示
    // 实现细节很容易变动 不应该被依赖
    // 应该是一个通知 用一个抽象的通知来代替这个具体的通知
    ProgressBar* m_progressBar;
public:
    FileSplitter(const string& filePath, int fileNumber, ProgressBar* progressBar) :
        m_filePath(filePath), 
        m_fileNumber(fileNumber),
        m_progressBar(progressBar) { }
    void split(){
        // 1.读取大文件

        // 2.分批次向小文件中写入
        for (int i = 0; i < m_fileNumber; i++){
            // ...
            float progressValue = m_fileNumber;
            progressValue = (i + 1) / progressValue;
            m_progressBar->setValue(progressValue);
        }
    }
};
```

上面示例中 在界面和文件分割器中都加了个进度条 传入进度条到文件分割器 给文件分割器去修改进度 违背了 依赖倒置原则 绑死了那个进度条(将来想换很麻烦)

```c++
class IProgress {
public:
    virtual void DoProgress(float value) = 0;
    virtual ~IProgress() { }
};
class FileSplitter {
    string m_filePath;
    int m_fileNumber;

    List<IProgress*> m_iprogressList; // 抽象通知机制 支持多个观察者
public:
    FileSplitter(const string& filePath, int fileNumber) :
        m_filePath(filePath), m_fileNumber(fileNumber) { }

    void split() {

        // 1.读取大文件

        // 2.分批次向小文件中写入
        for (int i = 0; i < m_fileNumber; i++) {
            // ...
            float progressValue = m_fileNumber;
            progressValue = (i + 1) / progressValue;
            onProgress(progressValue); // 发送通知
        }
    }

    void addIProgress(IProgress* iprogress) {
        m_iprogressList.push_back(iprogress);
    }
    void removeIProgress(IProgress* iprogress){
        m_iprogressList.remove(iprogress);
    }
protected:
    virtual void onProgress(float value) {
        List<IProgress*>::iterator itor = m_iprogressList.begin();
        while (itor != m_iprogressList.end() ) {
            (*itor)->DoProgress(value); // 更新进度条
            itor++;
        }
    }
};
class MainForm : public Form, public IProgress {
    TextBox* txtFilePath;
    TextBox* txtFileNumber;

    ProgressBar* progressBar;
public:
    void Button1_Click() {
        string filePath = txtFilePath->getText();
        int number = atoi(txtFileNumber->getText().c_str());

        ConsoleNotifier cn;

        FileSplitter splitter(filePath, number);

        splitter.addIProgress(this); // 订阅通知
        splitter.addIProgress(&cn); // 订阅通知

        splitter.split();

        splitter.removeIProgress(this);
    }
    virtual void DoProgress(float value) {
        progressBar->setValue(value);
    }
};
class ConsoleNotifier : public IProgress {
public:
    virtual void DoProgress(float value) {
        cout << ".";
    }
};
```

将进度展示类做成抽象(通知者) 文件分割器修改 IProgress 的进度(使用文件分割类里的通知者去发送通知) 发送通知以后 观察者就会去修改主界面的进度(调用继承自通知者并重写的 DoProgress 函数)

这里还可以将 addIProgress removeIProgress onProgress 单独抽成一个抽象类 做成通知基类 需要做成通知者的类就可以直接继承自这个基类

流程大致为
```
        抽象的通知者      -> 只留了个通知的接口
    virtual doProgress()
            A
            |
        overwrite
            |
            V
        具体的通知者  -> 实现了通知的接口具体做什么(可以理解为直接执行了真正想做的事情 
    overwrite doProgress()  又可以理解为是因为通知者发送了通知才执行了想做的事情)
```
