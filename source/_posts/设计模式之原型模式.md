## Prototype
属于 对象创建 模式

用于 隔离类对象的使用者 和 具体类型(易变类) 同样要求 易变类拥有稳定的接口 特别是用于 对象结构 特别复杂的情况... 如果不复杂的话 那就直接用 工厂方法 完事了

使用 原型实例 指定创建对象的种类 然后通过 深拷贝 原型来创建新的对象

下面的原型模式的实现是 根据之前我的 [设计模式之 工厂方法](https://www.yuerer.com/设计模式之工厂方法/) 这篇文章下的最后一个版本改的

```c++
// 可以看出 ISplitter 和 SplitterFactory 合并起来
// 因为 Prototype 创建对象不再是使用工厂去创建
// 而是使用自身的拷贝 因此这个设计模式 叫做 原型 是克隆的原型
class ISplitter {
public:
    virtual void split() = 0;
    // 通过克隆自己来创建对象
    virtual ISplitter *clone() = 0;
    virtual ~ISplitter() { }
};
class BinarySplitter : public ISplitter {
public:
    virtual ISplitter* clone() {
        return new BinarySplitter(*this);
    }
};
class TxtSplitter : public ISplitter {
public:
    virtual ISplitter* clone() {
        return new TxtSplitter(*this);
    }
};
class MainForm : public Form {
    // 原型对象
    ISplitter *prototype;
public:
    MainForm(ISplitter *prototype) {
        this->prototype = prototype;
    }
    void Button1_Click() {
        // 克隆原型 来避免使用工厂方法时创建对象太过于麻烦的问题
        ISplitter *splitter = prototype->clone();
        splitter->split();
    }
};
```

原型模式和工厂方法那么类似 那么应该选择哪一个设计模式呢? 最简单的方法就是 判断 你用工厂方法 创建一个对象 是不是只需要简单的几步就能创建出来? 如果不是 那就用原型模式
